 🎨 Crafty Portal — Документация учебного проекта

## 1. Название

**Crafty Portal** — информационная система управления творческим контентом и социальной интерактивности.

## 2. Краткое описание

Проект представляет собой полнофункциональное веб-приложение (SPA), разработанное как ключевой модуль ВКР. Система предназначена для организации единого пространства обмена творческим опытом: публикации авторских работ, ведения дискуссий через систему комментариев и управления портфолио.

### Основной функционал

- Регистрация и аутентификация пользователей (JWT).
- Загрузка и хранение медиаконтента (изображений) с метаданными.
- Публикация работ с уникальными описаниями.
- Система обратной связи (комментарии) под каждой публикацией.
- Личный кабинет автора с возможностью удаления собственного контента.

## 3. Системные требования

- **Python** 3.10+
- **Node.js** 18+ и **npm**
- **СУБД:** SQLite (встроена в Python)
- **Библиотеки Python:** FastAPI, SQLAlchemy, Uvicorn, Python-Multipart
- **Библиотеки JS:** React 18+, Vite, Axios, React Router 6+

## 4. Инструкция по установке

### Шаг 1. Клонирование

```bash
git clone <URL-вашего-репозитория> crafty-portal
cd crafty-portal
```

### Шаг 2. Бэкенд (FastAPI)

Из корневой директории проекта:

```bash
cd backend
python -m venv venv
```

**Активация виртуального окружения**

- Windows: `venv\Scripts\activate`
- Linux / macOS: `source venv/bin/activate`

**Установка зависимостей**

```bash
pip install fastapi sqlalchemy uvicorn python-multipart "python-jose[cryptography]" "passlib[bcrypt]"
```

База данных SQLite (`app.db`) будет создана автоматически при первом запуске приложения (если в `database.py` настроен драйвер SQLite).

### Шаг 3. Фронтенд (React)

В **новом терминале** из корня проекта:

```bash
cd frontend
npm install
```

## 5. Инструкция по запуску (локально)

### Запуск API (бэкенд)

Из папки `backend` (с активным `venv`):

```bash
uvicorn app.main:app --reload
```

- API: [http://127.0.0.1:8000](http://127.0.0.1:8000)
- Интерактивная документация (Swagger): [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

### Запуск интерфейса (фронтенд)

Из папки `frontend`:

```bash
npm run dev
```

Приложение откроется по адресу [http://localhost:5173](http://localhost:5173). Запросы к API настраиваются в `frontend/src/api.js` (базовый URL по умолчанию указывает на `http://127.0.0.1:8000`); для кросс-доменных запросов на бэкенде включён CORS.

## 6. Структура проекта

### Бэкенд

**Стек:** Python 3.10, FastAPI, SQLAlchemy (ORM), SQLite.

**Назначение:** REST API для обработки данных пользователей, хранения путей к изображениям и управления комментариями.

**Структура API** (префикс `/api` или прямой доступ к маршрутам роутеров — в зависимости от конфигурации; в типичной сборке используются пути без общего префикса `/api`):

| Метод  | Путь                         | Описание                                      | Доступ       |
|--------|------------------------------|-----------------------------------------------|--------------|
| `POST` | `/auth/register`             | Регистрация нового автора                     | Всем         |
| `POST` | `/auth/login`               | Вход, получение JWT-токена                    | Всем         |
| `GET`  | `/auth/me`                    | Данные текущего профиля (ID, имя)             | Bearer Token |
| `GET`  | `/works/`                     | Получение списка всех работ                   | Всем         |
| `GET`  | `/works/{id}`                 | Детальная страница работы + комментарии       | Всем         |
| `POST` | `/works/`                     | Загрузка изображения и создание поста         | Bearer Token |
| `DELETE` | `/works/{id}`               | Удаление работы (только автором)            | Bearer Token |
| `POST` | `/works/{id}/comments`       | Написать комментарий к работе                 | Bearer Token |

### Фронтенд

**Стек:** React 18, Vite, React Router 6, Axios.

**Назначение:** Single Page Application для взаимодействия с пользователем: визуализация галереи, динамические формы и управление состоянием.

**Основные маршруты и модули**

| Маршрут            | Файл            | Назначение |
|--------------------|-----------------|------------|
| `/`                | `Home.jsx`      | Глобальная лента творческих работ |
| `/login`, `/register` | формы в соответствующих страницах | Аутентификация |
| `/upload`          | `Upload.jsx`    | Интерфейс загрузки медиафайлов |
| `/works/:id`       | `WorkDetail.jsx`| Просмотр работы, комментарии, удаление для владельца |

**Ключевые файлы и каталоги**

- `frontend/src/api.js` — централизованная настройка Axios (`baseURL`, перехватчики токенов).
- `frontend/src/App.jsx` — маршрутизация и глобальный лейаут.
- `backend/static/uploads/` — директория для физического хранения загруженных файлов (относительно папки `backend` при запуске из неё).

## 7. Проверка функционала

1. Откройте главную страницу (лента должна быть пуста или содержать тестовые данные).
2. Пройдите регистрацию и войдите в систему.
3. Нажмите **«+ New Post»**, выберите изображение и добавьте описание.
4. Кликните на созданную работу — откроется детальный вид.
5. Напишите комментарий и проверьте, что он отображается с указанием даты.
6. Убедитесь, что на **вашей** работе видна кнопка **«Delete Post»**, а на чужих работах она отсутствует.
